This part finishes the learning path with boundary and workflow tools.
Attributes, foreign declarations, quote, testing, and command workflow all matter, but they matter most after ordinary Musi code already feels stable.
These pages should feel like practical extensions, not sudden genre change.

## What

This section teaches metadata, host-boundary declarations, code-as-data tools, testing surface, and everyday command workflow.
Each topic matters, but none of them should replace the simpler habits from earlier chapters.

## Why

If advanced topics appear too early, beginners get overwhelmed and leave.
If they never appear with concrete examples, later-stage users still have to ask how Musi handles tooling, native integration, or metaprogramming.
This section answers those needs without turning into a wall of detail.

## How

Treat these chapters as boundary tools.
Reach for them when ordinary code is already clear and you need metadata, external integration, generated syntax, verification, or repeatable workflow.
Keep the examples grounded in practical use so the advanced surface feels purposeful rather than ornamental.
