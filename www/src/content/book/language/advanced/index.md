This part finishes the learning path with boundary and workflow tools.
Attributes, foreign declarations, unsafe FFI, quote, testing, and command workflow all matter, but they matter most after ordinary Musi code already feels stable.
These pages should feel like practical extensions, not sudden genre change.

## In this chapter

This section teaches metadata, host-boundary declarations, unsafe FFI boundaries, code-as-data tools, testing surface, and everyday command workflow.
It distinguishes attribute families explicitly, so advanced pages answer more than "yes, attributes exist".

## Why it matters

If advanced topics appear too early, beginners get overwhelmed and leave.
If they never appear with concrete examples, later-stage users still have to ask how Musi handles metadata, native integration, generated syntax, verification, or repeatable workflow.
This section answers those needs without turning into a wall of detail.

## Walk through it

Treat these chapters as boundary tools.
Reach for them when ordinary code is already clear and you need metadata, external integration, generated syntax, verification, or repeatable workflow.
Follow visible example sources when one chapter shows boundary syntax you want to trace back into repo code.
