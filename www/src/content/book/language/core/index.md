This part turns first-file syntax into everyday working code.
You will move from direct values into operators, ranges, functions, calls, and methods, always with examples that look like real Musi rather than token drills.
By the end, small expression-oriented code should feel normal instead of novel.

## What

This section teaches core surface you read all the time: direct values, value transformations, reusable functions, ordinary calls, and receiver-led methods.
These are not isolated grammar facts; they are building blocks for nearly every later chapter.

## Why

Readers need enough substance here to stop asking basic "how do I compute this?" or "how do I call that?" questions later.
At same time, this section should not become a full reference manual before users can even read a tiny file comfortably.
The right balance is concrete examples with one clear lesson per chapter.

## How

Work through chapters in order and keep rewriting examples in your own words.
Name intermediate results, trace values left to right through calls, and compare plain functions with methods so you can choose the clearer style in real code.
If one form still feels mechanical, keep it small until the reading pattern becomes boring.
