Musi is easiest to learn when each chapter answers a real question with real code.
This book keeps the What, Why, How frame, but the goal is not tiny stub sections.
The goal is compact pages that show enough Musi to help beginners act, while still stopping before the material turns into an overwhelming wall.

## What

This book teaches current Musi from installation through files, expressions, data, types, abstractions, effects, runtime layering, and tooling.
Each chapter focuses on one concept, but it should still leave you able to write or read something concrete afterward.

## Why

If learning pages say too little, readers leave with unanswered "how do I ...?" questions and need external help immediately.
If pages say too much, readers drown in detail before the core model becomes stable.
This book aims for usable middle ground: one concept at a time, enough code to make it real, and enough explanation to show when the concept matters.

## How

Read in order when Musi is new.
Type the examples into scratch files, keep asking what each page's code is doing, and move forward once the chapter's main decision feels clear.
When you come back later, use the chapter order as map: start, core, data, organization, types, abstractions, effects/runtime, then advanced workflow.
